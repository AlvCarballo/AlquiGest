Error al exportar Excel

Guía paso a paso para resolver el problema de descarga de archivos .xlsx en AlquiGest.

Síntomas del error

Cuando el problema está presente, al pulsar cualquier botón de exportación Excel en AlquiGest ocurre una de estas cosas:

  • El navegador muestra un mensaje de error en texto plano en vez de descargar el archivo
  • Se descarga un archivo .xlsx pero Excel no puede abrirlo ("el archivo está dañado")
  • El servidor devuelve un error HTTP 500
🔍
Cómo ver el mensaje de error exacto

Abre las herramientas de desarrollador del navegador (F12), ve a la pestaña Red, haz clic en el botón de exportar y busca la petición a export.php. Haz clic en ella y mira la pestaña Respuesta.

Causa del problema

AlquiGest genera archivos Excel (.xlsx) usando la extensión ZipArchive de PHP, que construye el fichero zip internamente sin necesitar librerías externas. El problema es que esta extensión viene desactivada por defecto en XAMPP para Windows.

Solución en XAMPP (Windows)

Sigue estos pasos exactamente en el orden indicado:

1

Abre el Panel de Control de XAMPP

Busca el icono de XAMPP en la barra de tareas o en el escritorio y ábrelo. Verás los servicios Apache, MySQL, etc.

2

Selecciona "PHP (php.ini)"

En el menú desplegable del botón Config de Apache, selecciona PHP (php.ini). Se abrirá el archivo de configuración de PHP en el Bloc de notas o en tu editor de texto predeterminado.

3

Busca la línea de la extensión zip

Dentro del Bloc de notas, pulsa Ctrl + F y busca el texto extension=zip. Encontrarás la línea con un punto y coma al inicio, que la desactiva:

php.ini — antes Buscar esto 
;extension=zip
💡
El punto y coma es un comentario

En php.ini, las líneas que empiezan con ; están desactivadas (comentadas). Quitarlo activa la extensión.

4

Elimina el punto y coma y guarda

Borra el ; del inicio de la línea. Debe quedar exactamente así:

php.ini — después Resultado final 
extension=zip

Guarda el archivo con Ctrl + S. Si el sistema pide permisos de administrador, acéptalos.

5

Reinicia Apache en el Panel de XAMPP

Vuelve al Panel de Control de XAMPP. Pulsa el botón Stop junto a Apache y luego Start. Los cambios en php.ini solo se aplican al reiniciar el servidor.

⚠️
Es obligatorio reiniciar Apache

Si no reinicias Apache, php.ini no se recarga y el cambio no tendrá efecto.

Solución en MAMP

En MAMP la extensión ZipArchive suele estar activa por defecto. Si aun así falla, comprueba en MAMP → Preferencias → PHP la versión de PHP seleccionada y busca en el php.ini correspondiente.

La ubicación del php.ini en MAMP (Mac) suele ser:

Terminal Mac
/Applications/MAMP/bin/php/phpX.X.X/conf/php.ini

En MAMP para Windows:

Ruta Windows
C:\MAMP\bin\php\phpX.X.X\php.ini

Verificar que la solución funciona

1

Comprueba ZipArchive con phpinfo()

Crea un archivo llamado test.php en la carpeta de AlquiGest con este contenido:

test.php — temporal
<?php
if (class_exists('ZipArchive')) {
    echo '✅ ZipArchive ACTIVADO — La exportación Excel funcionará.';
} else {
    echo '❌ ZipArchive NO está disponible. Revisa los pasos anteriores.';
}
phpinfo();

Accede a http://localhost/AlquiGest_v3/test.php en el navegador. Si ves el mensaje verde, está solucionado. Elimina test.php después de la prueba.

2

Prueba la exportación en AlquiGest

Vuelve a AlquiGest, ve a la pestaña de Informes y pulsa cualquier botón de exportación Excel. El archivo se debería descargar correctamente.

✅
Solución completada

Si el archivo .xlsx se descarga y Excel puede abrirlo, el problema está resuelto.

Otros problemas comunes con la exportación

El archivo se descarga pero Excel dice "archivo dañado"

Esto ocurre cuando PHP emite algún aviso o mensaje de error antes de los bytes binarios del archivo Excel. Causas frecuentes:

  • PHP muestra un Warning o Notice al principio de la respuesta
  • El archivo php.ini tiene display_errors = On y PHP está mostrando errores en la salida

Solución: cambia en php.ini display_errors = Off y reinicia Apache.

Error de permisos al crear el archivo temporal

En algunos sistemas Windows, PHP no tiene permisos para escribir en la carpeta temporal. Comprueba que la carpeta devuelta por sys_get_temp_dir() existe y tiene permisos de escritura para el usuario del servicio Apache.

La extensión ZipArchive aparece en phpinfo() pero aun así falla

Asegúrate de que estás editando el php.ini correcto. XAMPP puede tener varios. Busca en phpinfo() el campo Loaded Configuration File para saber cuál está usando realmente.
